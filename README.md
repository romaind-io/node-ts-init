# node-ts-init

Struggling to initialize a clean Node.js + TypeScript project every single time? Me too. 
This bash script automates the tedious setup so you can start coding in seconds.

## 📋 Prerequisites

- `Bash` (or compatible shell)
- `Node.js` and `npm` installed

## 📦 Quick Start

In one command line:
```bash
curl -O https://raw.githubusercontent.com/romaind-io/node-ts-init/main/init.sh && chmod +x init.sh && ./init.sh
```
⚠️ Always review the script before executing it in your environment.

This command fetches the script from GitHub, grants execution permissions, and runs it. The script shoulds automatically remove itself after execution.

And you're free to code!

## </> Dev Environment

I personaly use `VSCode` and so it creates a `.vscode` config folder with `extensions.json` and `settings.json`.

### I recommend to use:
  - `Prettier - Code formatter` as a code formatter
  - `ESlint` to detect common problems in your code
  - `JavaScript and TypeScript Nightly` to get advanced Typescript things from `typescript@next`

### I setup VSCode to:
  - automatically formats code on save, removes unused variables/imports, and sorts imports.
  - setup a rulers to visualize the limit imposed by prettier code formater (80 by default)

## ✨ Prettier

`Prettier` is particularly useful when working on project in collaboration with other devs, it's here to keep the syntax logic between all the collaborators.

## 🔎 ESLint

`ESLint` is a great tool to prevent you from common JS/TS problem in syntax (esm / commonjs) and in the logic of your code.

## 🧪 Vitest

I use `Vitest` to test my files named `*.spec.ts`. Based on `Vite`, it's an efficient testing-framework, fast and customizable in its use.

## 🔑 .env

A `.env` file is included by default for environment variable management. You can remove it if not needed.

## 🟦 Typescript

I use `Typescript` daily and it's quite comforting to have this type safety in your project in order to prevent bugs. I've setup 3 configs:

- ### tsconfig.json
  - will define your project configuration
  - use to get a strict and strong-typed configuration (can be too verbose for some but I like it)

- ### tsconfig.build.json
  - use to build the project ignoring all the test files

- ### tsconfig.test.json-
  - use to recognize paths for import in test files

## 📜 NPM scripts

### 🔨 `build`
  This script allows you to build your project in a `build` folder.

### ✨ `format`
  This script use `Prettier` to rewrite the code in the project with the chosen setup.

### 🔎 `lint`
  This script use `ESLint` to correct bugs that it can detect and encounter.

### 🚀 `start`
  This script start the application generated by the `build` script. It will fail if none has been generated.

### ⚙️ `start:watch`
  This script use `tsx` to start your application from `.ts` file and listen to the change when you're coding.

### 🧪 `test`
  This script use `Vitest` to execute all the test files in your code.

### 🧪 `test:watch`
  This script use `Vitest` to execute all the test files in your code and listen to the change when you're coding.
